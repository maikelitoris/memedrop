Phase 1: Daily Meme Drop - Expanded Design Document

A minimalist Flutter mobile app where users receive two time-gated meme drops daily.

CORE CONCEPT AND PHILOSOPHY

The Vibe: Digital artifact hunting. Scarcity creates value. 
No FOMO through grinding - just pure, time-boxed serendipity.

Key Principles:
- Anti-engagement: The app resists infinite scroll
- Scarcity economics: Miss it, lose it forever
- Anonymous by design: You are your collection, not your profile
- Brutalist honesty: Raw textures, system fonts, no polish theater

USER JOURNEY AND FLOW

Entry Point: The Landing
Visual: Concrete gray background. Centered, massive typography: "DROP."
Interaction: Single tap anywhere triggers immediate generation of Anonymous ID (UUID v4) + timestamp
Storage: SharedPreferences key 'anonymous_id' (persistent across sessions)
No: Email fields, social auth buttons, "Continue with...", onboarding carousels

The Home Screen: The Vault
Primary Element: Sealed container (envelope/box/card-back) centered, 70% viewport
States:
- Sealed: Subtle breathing animation (scale 1.0 to 1.02 loop, 3s). No text label - visual language only
- Countdown: Micro-typography below container: "NEXT DROP: 04H 23M" (updates in real-time based on device clock vs last_opened timestamp)
- Ready State: When window opens, container emits faint glow + haptic pulse (if enabled)

Interaction: Tap container triggers reveal sequence

3. Rarity Shuffle (300-600ms): 3-5 meme thumbnails flash rapidly in center slot (slot machine effect)

TECHNICAL ARCHITECTURE

Time-Gating Logic (The Gatekeeper)

Pseudocode for drop validation:

```dart
bool canOpenDrop() {
  final now = DateTime.now();
  final lastOpened = prefs.getInt('last_opened_timestamp') ?? now;
  final firstWindowStart = DateTime(now.year, now.month, now.day, 0, 0); // 00:00 today
  final firstWindowEnd = firstWindowStart.add(Duration(hours: 12));
  final secondWindowStart = firstWindowEnd; // 12:00
  final secondWindowEnd = secondWindowStart.add(Duration(hours: 12));
  
  // User can ONLY open if:
  // 1. First time ever (no timestamp)
  // 2. Last opened was BEFORE current window started
  if (lastOpened == null) return true;
  
  // Check if we're in window 1 (00:00-11:59)
  if (now.isAfter(firstWindowStart) && now.isBefore(firstWindowEnd)) {
    // Must have last opened before window start
    return lastOpened.isBefore(firstWindowStart);
  }
  
  // Check if we're in window 2 (12:00-23:59)
  if (now.isAfter(secondWindowStart) && now.isBefore(secondWindowEnd)) {
    // Must have last opened before window start
    return lastOpened.isBefore(secondWindowStart);
  }
  // Else: Too late, wait for tomorrow
  return false;
}
```

CRITICAL IMPLEMENTATION DETAILS

Time Window Logic:
The key challenge: Preventing timezone manipulation (user changing device clock to cheat drops).

Solution:
- Store last_opened_timestamp in UTC
- Compare against DateTime.now().toUtc()
- Window calculation uses UTC day boundaries (00:00 UTC / 12:00 UTC)
- Local display converts UTC to device timezone for countdown

Edge Case Handling:
- App killed/backgrounded during reveal: Save State, then Resume animation on return
- Midnight boundary: If opened at 11:59:59, timestamp records that Window 1 was "consumed"
- Device clock changes: If user changes clock backward, last_opened remains honest (they don't get extra drops)

The "Stash" Action:
When user taps "STASH TO GALLERY":
1. Haptic confirmation (light)
2. Asset added to collection list in preferences
3. UI immediately updates (hero animation to gallery position)
4. No cloud upload. Local only.

Gallery Sort Order:
Default: Acquired (newest first)
Optional sorts: Rarity (Legendary to Common), Random shuffle

Memory Management:
- Collection stored as JSON string in SharedPreferences
- Limit: No enforced limit (phones have storage), but consider pagination if greater than 500 items
- Images cached via flutter_cache_manager or similar (load on demand)

The Countdown Display:

```dart
// Logic for "Next drop in Xh Ym"
String getCountdownText() {
  final now = DateTime.now().toUtc();
  final nextWindow = now.hour < 12 
      ? DateTime.utc(now.year, now.month, now.day, 12, 0)  // 12:00 UTC
      : DateTime.utc(now.year, now.month, now.day + 1, 0, 0);  // 00:00 UTC tomorrow
  
  final difference = nextWindow.difference(now);
  final hours = difference.inHours;
  final minutes = difference.inMinutes % 60;
  
  return "NEXT DROP: ${hours}H ${minutes}M";
}
```
